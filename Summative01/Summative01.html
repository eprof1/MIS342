Learning Objectives

Learning Objectives

  1. Understand the fundamental components of a database, including fields, records, and tables.
    • Example: In a customer database for a retail business, a field could represent individual data points like a customer's name or email address, a record would represent the complete set of information about one customer, and a table would store all customer records.
  2. Define and explain the significance of a primary key in a table.
    • Example: In an inventory database for a store, the primary key could be the Item ID, ensuring each product has a unique identifier, which helps avoid confusion when managing stock levels.
  3. Describe the purpose and function of queries, forms, and reports in Microsoft Access.
    • Example: A query could be used to retrieve a list of all customers who made purchases in the last month. A form allows employees to enter new sales transactions into the database efficiently, and a report can generate a summary of monthly sales figures for management review.
  4. Identify the key features and benefits of a Database Management System (DBMS).
    • Example: A DBMS allows a retail chain to manage customer data centrally, ensuring consistency and accuracy across all stores, and providing insights into purchasing trends for marketing strategies.
  5. Demonstrate the processes of compacting and backing up a database to ensure data integrity and recovery.
    • Example: Regularly compacting a sales database helps optimize performance by reducing file size, while backing up ensures that data is recoverable in case of hardware failure, protecting against data loss and ensuring business continuity.

Exercise 1-Tables

You have been hired by a local travel agency to set up tours for the coming season to the National Parks in the United States. The travel agency wants to be able to access a database that will let them know about several features of each park so they can best meet the needs of prospective travelers. As clients book and then take the trips, the database will also include client information that can be linked to the park information. Your job is to search the Internet for information about the National Parks and set up the National Parks database.

Start Access and open a new blank database. Name the database National Parks.
Use your favorite search engine to research information on the National Parks in the United States.
Using the https://www.nps.gov website, find or make up basic information on at least five parks.
Information can include the name of the park, the state in which the park is located, the size of the park in acres, adult admission fee in dollars, special features, and the date that the park opens for the season.
Create a table named Park.
Accept the default ID primary key field with the AutoNumber data type, renaming it as Park ID.
Enter five fields- Name, State, Fee, Hours, and Date Opened, in the table based on the information you found.
Enter other fields that you deem appropriate. Be sure to define the data type appropriate for each field and to use appropriate field naming techniques.
Enter at least five records in the Park table for the parks that you found during the Internet search.
